# C++ Live HTTP Server

Простой кроссплатформенный HTTP сервер на C++, который:

* Раздаёт `index.html` и `style.css`.
* Автоматически обновляет файлы при изменении (HTML и CSS).
* Работает на Windows и Linux.
* Поддерживает Ctrl+C для корректного завершения.

---

## Особенности

* **Авто-перезагрузка файлов**: сервер следит за изменениями и сразу отдаёт свежую версию при новом подключении.
* **Кэширование**: файлы загружаются в память для быстрого ответа.
* **Кроссплатформа**: работает и на Windows, и на Linux.
* **Простота использования**: не требует внешних библиотек.

---

## Файлы проекта

* `main.cpp` — исходный код сервера.
* `index.html` — HTML файл для отображения на сервере.
* `style.css` — CSS файл для стилизации страницы.

---

## Компиляция

### Linux

```bash
g++ main.cpp -o server -pthread
```

### Windows (Visual Studio / g++)

```bash
g++ main.cpp -o server.exe -lws2_32 -pthread
```

> На Windows необходимо подключать библиотеку `ws2_32` для работы с сокетами.

---

## Запуск

1. Убедитесь, что в папке находятся `index.html` и `style.css`.
2. Запустите сервер:

```bash
./server      # Linux
server.exe    # Windows
```

3. Откройте браузер и перейдите по адресу: [http://localhost:8080](http://localhost:8080)
4. Любые изменения в `index.html` или `style.css` будут автоматически применяться при обновлении страницы.

---

## Остановка сервера

* Нажмите `Ctrl+C` в терминале — сервер корректно завершит работу и освободит порт.

---

## Как работает

1. Сервер запускает сокет на порту 8080.
2. Все HTTP-запросы обрабатываются в основном цикле.
3. Отдельный поток каждые 0.5 секунды проверяет время последней модификации файлов `index.html` и `style.css`.
4. Если файл изменился — сервер перезагружает его в память (`html_cache` или `css_cache`).
5. При новом запросе сервер отдаёт актуальную версию файлов.
